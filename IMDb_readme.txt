Why should we structure HTML semantically for something like a login form? 
It is easier to keep track of the different elements by structuring HTML seamtically.

What are some benefits of everyone using similar semantic HTML? 
When working on the same project, the other team members can understand what is another team member trying to do. it make to job more coordinated.
The machine can also understand instructions better.

What are some disadvantages? 
Some semantic in HTML might become obsolete and if everyone used the same obsolete semantic, it would be hard to update the program.
